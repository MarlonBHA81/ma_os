# How ma_os works

The operating manual for this vault. Both Marlon and Claude follow these rules so the brain stays coherent as it grows.

## The four purpose areas (+ capture + maps)

- **[[Projects]]** — active efforts with a goal and an end (e.g. "Launch Q3 course"). When a project finishes, it moves to [[Archive]].
- **[[Areas]]** — ongoing responsibilities with no end date (e.g. "StoryAdvantage", "Health", "Finances").
- **[[Resources]]** — reference knowledge and topics of interest, independent of any project (e.g. "Storytelling frameworks", "Book notes").
- **[[Archive]]** — anything inactive. Nothing is deleted; it's archived.
- **[[Daily]]** — the inbox. Fast, raw capture lands here with a timestamp. Durable ideas get promoted into their own note and linked.
- **[[Maps]]** — Maps of Content: hub notes that gather links to related notes across the other folders. A Map is how you find things without tags.

## Notes are atomic and linked

- One idea per note. A note's **title is the idea** (descriptive, plain language) — that's how it gets found and linked.
- Link liberally. Any concept, person, project, or topic a note mentions should be a `[[wikilink]]`. Linking to a note that doesn't exist yet is encouraged — it marks something worth writing later.
- No tags. Folders say *what kind* of thing it is; links say *what it relates to*. We organize better as we go rather than up front.

## Minimal frontmatter

Only `created: YYYY-MM-DD`. Nothing else is required.

## Growing gracefully

When a folder or Map gets crowded, that's the signal to split it into a more specific Map or sub-topic — not before. Structure emerges from what's actually there.
