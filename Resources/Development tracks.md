# Development tracks

Stage 5 of the [[Story Advantage delivery process]]. The designed [[Wireframe page set]] gets built — on **one of three routes**, chosen per client:

1. **Divi / WordPress** — build out the WordPress + Divi layouts by hand.
2. **GHL (GoHighLevel)** — write a **prompt for GHL's AI builder** to generate the site/funnel.
3. **HTML** — hand-built, via `js.code`.

*(The [[BRS Scanner]] itself is a separate Next.js app — see [[BRS Scanner — live data]] — not one of these client-delivery routes.)*

Feeds into [[Launch and promo]] (stage 6) — where "build a promo" and go-live happen.
